## 第 1 层｜红沙腹地：一个持续变化的解谜世界

红沙腹地以绿洲为生活据点、以荒漠与洞穴为探索空间。玩家通过观察、测量、推理、规划和实验，解决居民、商队、生物与自然环境不断出现的现实问题，并让自己的方案持续改变世界。

最终主题是：

> 长久并不来自取之不尽，而来自观察、协作、节制和合理的循环设计。

### 第 2 层 A｜定位与体验：玩家为什么愿意留在这里

#### 第 3 层 A1｜玩家身份与区域目标

玩家不是围绕一件神器行动的“救世主”，而是逐渐理解红沙腹地运行规律的调查者、建设者和问题解决者。

- 近期目标：解决眼前的缺水、迷路、沙暴、设施故障和生物异常。
- 中期目标：恢复绿洲、打开安全路线、勘明洞穴、建立居民关系。
- 长期目标：让水网、商路、地下生态和聚落生活能够共同运转。
- 长明灯定位：它是地下遗迹线中的一个重要章节，用来改善夜行与生态；不是整片区域的唯一目标。

#### 第 3 层 A2｜生活、远行、洞穴三种体验

- 绿洲生活：休整、认识居民、陪伴骆驼、修复设施、观察世界变化。
- 荒漠远行：规划路线、带水、安排负重、判断沙暴、选择避难点。
- 洞穴调查：利用回声、水位、地势、遗迹与生态痕迹推理深层原因。

三种体验的节奏是：先在绿洲获得安全感，再在荒漠承担轻度压力，最后在洞穴集中思考；完成后回到绿洲，看到成果真正留在世界里。

#### 第 3 层 A3｜问题驱动与世界反馈原则

- 先让世界出现问题，再让玩家发现数学关系；不直接派发“计算题”。
- 所有思考都要转化为操作，例如量水、标记、装载、搭建和推进时间。
- 大多数问题允许多种方案，比较的是成本、风险、长期收益和对不同角色的影响。
- 错误方案必须产生可读反馈，例如水流到错误位置、沙暴先封路、骆驼因超载减速。
- 任务完成后，水面、植物、道路、商队、居民行为或可用路线必须发生持续变化。

### 第 2 层 B｜核心玩法：玩家每次如何解决问题

#### 第 3 层 B1｜单次任务循环

**发现异常 → 调查取证 → 提出猜想 → 制定方案 → 准备并执行 → 观察结果 → 世界变化**

- 发现异常：例如农田干枯、水渠傍晚断流、商队未归、风塔停转、洞穴动物迁移。
- 调查取证：与居民交流，并收集水量、脚印、影子、温度、震动、路线等证据。
- 提出猜想：通过选择继续调查哪里、携带什么、先修什么来表达判断。
- 制定方案：比较不同方案的代价、风险与长期结果。
- 准备并执行：规划路线、水和负重；抵达现场后调整、分配、搭建或试验。
- 观察结果：让世界演示因果，并把新证据记录到手册中。

#### 第 3 层 B2｜长期成长循环

**解决问题 → 改变区域 → 出现新需求 → 获得新工具与路线 → 解决更复杂的问题**

一个典型因果链如下：

- 修复风塔，抽水增加。
- 夜光草能够扩大种植。
- 特殊昆虫回归，地下入口显现。
- 古代蓄水设施开放，新水源吸引商队。
- 商队增加后，新的避难点、道路和分水问题又会出现。

#### 第 3 层 B3｜五套通用操作

- O1 调查测量：量水罐、秤、测杆、绳尺、温度计与环境敲击；用于水渠、脚印、假秤、井中刻度、回声和震动。
- O2 记录比较：在证据册中排序、对照、叠图、查看趋势与周期；用于水量、影子、月相、温度、星图和单位差异。
- O3 地图规划：连接地点，叠加风险图层，预览时间、补给和覆盖范围；用于商路、沙暴救援、避难屋和洞穴路线。
- O4 资源配给：用统一拖放板处理重量、容量、最低需求、价值、平衡和交换比例；用于分水、装货、交易和燃料。
- O5 工程调整：放置、旋转、连接少量模块，再推进时间观察结果；用于水渠、风塔、断桥、遮阳墙、挡沙墙、支架和镜光装置。

全部操作共用同一套反馈：**执行方案或推进时间 → 世界演示结果 → 手册出现新证据 → 玩家修改方案。**

### 第 2 层 C｜世界系统：哪些规则让世界持续运转

#### 第 3 层 C1｜绿洲水网

水是整个区域的主资源，也是任务之间最重要的连接点。

- 玩家测量不同地点、不同时间的水量。
- 玩家调整闸门、修补水渠、改变流向、使用蓄水池或风塔。
- 玩家在居民、医馆、骆驼、种植、树木和商队之间设定供水优先级。
- 水网结果直接影响绿洲设施、植物、居民活动和道路开放状态。

首版不做全实时水文模拟。季节、蒸发、堵塞和地下渗漏只作为具体任务规则或阶段修正出现。

#### 第 3 层 C2｜荒漠行程与骆驼伙伴

远行只长期保留三个决策变量：

- 时间：能否在日落、沙暴或期限前抵达。
- 水：能否完成路线并留下应急余量。
- 负重：携带物资如何影响速度、耐力和可通行道路。

骆驼是伙伴，不是单纯坐骑：

- 成长方向：负重、耐力、抗沙暴。
- 陪伴方式：喂食、梳理、训练和共同远行。
- 玩法价值：探测被掩埋路径、帮助选择路线、承接负重取舍。
- 失败恢复：路线判断错误时，可以前往避难点、丢弃非必要货物或回到绿洲重新规划，不造成突然死亡。

#### 第 3 层 C3｜据点建设与区域状态

据点建设采用有限设施节点，而非完全自由的建造模拟。

- 可修复设施：水渠、蓄水池、风塔、避难屋、挡沙墙、遮阴设施、夜间引导灯和镜光装置。
- 每项设施必须同时带来视觉变化、区域状态变化和后续任务的新条件。

区域长期结果收敛为三条可见状态：

- 水源稳定度：从断流与争水，到稳定供水与植被恢复。
- 道路安全度：从商路中断与夜行危险，到路线开放与避难点完善。
- 聚落与生态活力：从生物减少、集市萧条，到商队增多、居民活动丰富。

每次任务主要改变其中一项，最多附带影响另一项，避免状态组合过多而难以理解。

### 第 2 层 D｜任务内容：玩家具体会遇到哪些问题

#### 第 3 层 D1｜三级任务结构

- 区域主线：3 条，每条 4 章；负责改变水网、商路和地下生态等大状态。
- 角色支线：首发 6 条，每条分两次相遇；负责塑造人物、深化操作、提供替代方案。
- 日常与动态事件：5 个参数化模板；负责让种植、交易、伙伴和环境持续产生生活感。

主线负责“大变化”，支线负责“人与职业”，日常负责“区域持续活着”。支线不会成为主线硬门槛。

#### 第 3 层 D2｜主线 A：绿洲水网

目标：从每天被动救火式分水，转向可持续、可调整的供水网络。

- A1《会消失的第三条水渠》：测量流入和流出，判断水是被偷、蒸发还是漏入地下；解锁水网图和量水工具。
- A2《风塔与错误刻度》：解决井壁刻度、测量绳和风塔数据的矛盾；恢复风塔并提高数据可信度。
- A3《旱季该把水给谁》：在居民、医馆、骆驼、农田、树木和商队之间分配有限用水；决定即时生存、外部补给或长期固沙的优先级。
- A4《绿洲的新水路》：综合前述数据选择修渠、蓄水、改道或湿地方案；形成不同倾向的绿洲状态。

#### 第 3 层 D3｜主线 B：移动沙丘与商路

目标：从被动躲避沙暴，转向预测风险、建设设施和维持安全商路。

- B1《失踪的驼铃》：根据脚印、步距、铃声和风向寻找运药骆驼；获得骆驼伙伴与追踪能力。
- B2《比沙暴先到》：在沙暴前救出商队，并选择水、药物、绳索与货物的装载方案。
- B3《三座避难屋》：在有限材料下选择避难屋位置；让居民、商队或洞穴探索者获得不同便利。
- B4《被沙吞没的村庄》：根据沙丘、风向和植被数据选择挡风、固沙、搬迁或改道；改变村庄布局和商路安全。

#### 第 3 层 D4｜主线 C：地下遗迹与生态

目标：勘明洞穴、建立安全通路，并恢复地下设施与生态。

- C1《十二声回响》：依据回声和高度绘制多层洞穴，识别“地图相交但道路不相通”的陷阱。
- C2《洞穴里的震动》：根据震动、落石与动物迁移判断危险源，选择加固、封闭或替代通路。
- C3《墙上的星星多了一颗》：旋转、缩放并叠合星图，找到地下水网控制室；在恢复水网与保护生态之间选择优先级。
- C4《长明灯与镜光长廊》：调查夜光草、昆虫周期与可持续燃料量，用少量灯油和镜片恢复照明；选择强光商路、低耗生态照明或折中方案。

长明灯的结论是：灯油只能启动系统，真正长期的照明来自光路、植物、昆虫和水汽形成的循环。

#### 第 3 层 D5｜角色支线与日常事件

角色支线：

- 井务记录员《刻度之争》：理解单位、绳长、井形和记录误差。
- 药圃农户《没有影子的菜园》：利用移动阴影种植药草。
- 老牧民《骆驼不愿走的路》：分辨负重、疲劳和恐惧造成的拒行。
- 失忆商人《没有地图的商人》：把太阳、步数、地标和声音转换成路线。
- 集市秤匠《假秤与三种盐价》：比较秤臂、单位价格、含水量和纯度。
- 洞穴研究者《会同时开花的夜光草》：追踪温度、月相、浇水日与昆虫周期。

日常事件：

- 晨间水务：快速或深入地处理分水问题，后期可委托。
- 家园与伙伴照料：整理储备、安排阴影、照顾骆驼与少量种植。
- 商队出发板：结合天气、道路和货物安排远行。
- 月相集市：处理周期、单位、库存和商品质量。
- 环境巡检：通过风向、水位、震动和痕迹预测沙暴、地震与迁徙。

### 第 2 层 E｜成长与反馈：玩家如何变强、如何看见成果

#### 第 3 层 E1｜四阶段成长

- 第一阶段｜在绿洲安顿：学习量水、标记与基础分水；获得骆驼和稳定据点。
- 第二阶段｜学会荒漠行动：学习路线、时间、负重和避难；扩大安全活动范围。
- 第三阶段｜调查深层原因：学习回声、空间、数据可信度、规律与证据链；打开地下路线。
- 第四阶段｜参与区域治理：处理比例、多目标权衡、公平和长期收益；形成玩家选择塑造的区域状态。

完成第一阶段后，三条主线可以交错推进，避免玩家被单一线性叙事束缚。

#### 第 3 层 E2｜分层思考与提示

- 直觉层：用颜色、图标、沙漏、实际倒水和结果动画完成基础目标。
- 分析层：读取数字、比较路线、记录数据，用更少的试验完成任务。
- 优化层：同时考虑负重、备用路线、长期收益和多个对象，争取更优结果。

提示从轻到重依次提供：未观察的证据类型、已有数据的整理、一次方案预演、接近完整的操作建议。提示不会直接公布算式答案。

#### 第 3 层 E3｜奖励、失败与数学思维

奖励优先扩大选择空间：

- 工具升级：更准确的量具、地图叠图、单位换算与自动记录。
- 伙伴成长：更稳负重、更长耐力、沙暴中识路。
- 设施升级：蓄水池、遮阳棚、风塔、避难屋和引导灯。
- 人物关系：得到情报、工具、替代方案和新的合作方式。
- 世界反馈：植被恢复、商队增加、生物回归、庆典出现。

失败不直接判错：玩家可以从水位、路径、负重和结构变化中读到原因，再重置、回收资源或采用替代路线。

数学思维通过水量与交易承载数量和比例，通过测量和记录承载数据可信度，通过洞穴、影子和镜光承载空间几何，通过月相和生物承载规律周期，通过路线与设施承载优化和系统思维。

### 第 2 层 F｜首版落地：先做什么、暂时不做什么

#### 第 3 层 F1｜垂直切片目标与范围

首个切片只验证三件事：

- 玩家是否觉得“发现问题—调查—准备—解决—世界改变”的循环有趣。
- 多个任务能否复用同一套操作，而不变成独立小游戏。
- 水和高温构成的轻生存压力，是否促进取舍而不打断解谜。

切片范围：

- 一个小型绿洲聚落。
- 一段有三处明确地标的红沙路径。
- 一个包含三步连续环境谜题的小型洞穴。
- 一只骆驼、一项主动探测能力、三名居民、一支商队和两种非战斗生物。
- 一条“绿洲供水异常”主问题链，外加“受困商队”和“长明灯”两个可选问题。

预计完整体验时长：45—60 分钟。

#### 第 3 层 F2｜切片体验流程

- 玩家看到绿洲水量下降，发现居民、骆驼和商队受到不同影响。
- 玩家在三处量水点收集证据，确认第三条水渠出现异常。
- 玩家规划短途路线和骆驼负重，可先帮助商队、修复途中设施或进入洞穴。
- 玩家在洞穴中复用测量、记录、配给和工程操作，恢复地下水路。
- 长明灯作为可选小问题，复用风道、遮挡、燃料和镜片规则，不增加专属小游戏。
- 返回绿洲后，水面、植被、居民位置、商队状态和补给出现可见变化。

#### 第 3 层 F3｜验收门槛、边界与风险控制

进入扩产前，应同时满足：

- 至少 80% 的首次测试玩家能在 5 分钟内说清当前问题和下一步目标。
- 至少 70% 的玩家能在 60 分钟内完成主问题链。
- 至少 70% 的任务交互由共享组件完成。
- 没有关键流程软锁，谜题可以快速重置或恢复。
- 至少 70% 的玩家能注意到绿洲变化，并解释它与自己方案的关系。

首个切片暂时不做：

- 无缝大世界、多绿洲、战斗、敌人 AI、Boss。
- 完整昼夜、季节、天气、经济和生态模拟。
- 饥饿、疾病、睡眠、装备耐久、复杂制作配方。
- 多伙伴编队、繁殖、复杂技能树和完全自由建造。
- 程序化任务、大量分支对话、多阵营、多结局。
- 只服务单个任务的专属小游戏、专属 UI 或独立资源条。

所有新增内容采用“替换制”：加入一项新工作量，就删除或替换一项同等工作量内容，避免范围持续膨胀。

## 评审时只需要依次回答六个问题

1. A｜这个区域想让玩家体验什么？
2. B｜玩家反复在做什么？
3. C｜哪些系统保证世界会持续变化？
4. D｜主线、支线和日常内容各自承担什么？
5. E｜玩家为什么会继续成长和投入？
6. F｜第一版到底做什么，如何判断它值得继续扩展？

如果六个问题都能被清楚回答，这份策划案就具备了继续进入关卡拆解、美术概念和原型开发的基础。
